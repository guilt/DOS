About shared resources
======================
Shared resources are the disk drives, CD-ROM drives, 
printers and other devices attached to a server. All of 
these devices have the potential of being shared on the 
network. You just have to give them a network name, so 
LANtastic can identify them and make connections among 
them. Giving a device a network name is known as "creating 
a shared resource."

When you installed the LANtastic software, a printer 
resource named @PRINTER was automatically created on each 
server. A named resource was also created for every hard 
disk, floppy drive, disk partition and CD-ROM that 
LANtastic detected on the server. To see a list of these 
default shared resources on a server, choose Shared 
Resources Management from the LANtastic NET_MGR main 
screen. Each shared resource in the list has the same name 
that it uses on your system - for example, C-DRIVE. For a 
typical list of shared resources, see the Resources window 
shown under the "Creating a shared disk or directory 
resource" later in this file.

If the default resources listed in the server's Shared 
Resources Management screen are the only resources you 
want to share on the network, you don't have to create any 
more resources. However, if you want to limit access to 
certain directories on the server's hard disk, you'll need 
to use LANtastic to set up each directory as a separately 
named shared resource. You'll also have to set up a new 
printer or drive resource if you buy a new printer or 
CD-ROM drive that you want to share, or if a server has 
more than one printer that you want to share on the 
network. 

Planning your drive resources
=============================
As discussed in the previous section, during installation 
LANtastic created a resource for each drive that it 
detected on your server. For security reasons, you may 
want to change this default setup that gives everyone full 
access to every drive, including the server's C: drive. 

One method for doing this is to create a set of login 
accounts on the server. That way only a person with a 
valid username and password can log in to it and use its 
drives. For instructions, see "Creating a new login 
account" in the file Accts.txt. 

Once you've created a set of login accounts on the server, 
you can set Access Control List (ACL) rights for them. 
ACLs give an individual account, wildcard account, ACL 
group or template such as ACCT-* varying levels of access 
to a single resource on the server. For example, you could 
give one group of accounts only Read Access to a directory 
on C: drive, but allow another group both Read and Write 
Access to it. This type of setup is shown in the second 
diagram near the end of this section. 

For a list of ACL rights, refer to "Setting ACL rights for 
drive resources" later in this file. For instructions on 
setting up separately accessed directory resources, see 
"Creating a shared disk or directory resource" later in 
this file. For instructions on setting up ACL rights to a 
file, see the "Setting file-level security" section later 
in this file.

Note - If you're concerned about control directory 
security, don't give users any access to a resource that 
contains a control directory. You may also want to 
consider establishing control directory password 
protection, as described in the file Crldir.txt. For a 
listing of additional server security strategies, see the 
file Mngsrv.txt.

In addition to setting up secure directories of data 
files, you may want to create directories for shared 
applications. For example, if everyone in the accounting 
department will be running the same application program, 
you can set up a network version of it in a special 
directory on the server's C: drive. That way all of your 
accounting personnel can run it off the same drive, and 
you don't have to buy a copy for each computer. Make sure, 
however, that you're using a network version of the 
software and that sharing the program doesn't conflict 
with any licensing agreements.

Be sure to plan the organization of your directories and 
subdirectories before you start setting them up as shared 
resources.

The ACL rights set for a shared directory also apply to 
all of its subdirectories. You can use file-level security 
to further limit access. See "Setting file-level security" 
later in this file for details.

Creating a shared disk or directory resource
============================================
In order for users to access disk drives, subdirectories 
and printers on a server, you must first set up the 
server's physical resources (hard drives and printers) as 
shared resources using LANtastic's NET_MGR program.

1. At a LANtastic computer's DOS prompt, type NET_MGR 
<Enter>. The NET_MGR main screen appears. 

2. Select Shared Resources Management <Enter>. A screen 
appears showing all the server's resources. 

3. Press Ins and type a name that will identify the shared 
resource for the user <Enter>. This name may be up to 
eight characters long, with a three-character extension. 
For example, if you want to set up your C: drive as a 
resource, you might give it the name C-DRIVE. 

4. When NET_MGR asks for the true path, type in the local 
name of the server disk drive or directory <Enter>. For 
example, to set up your C: drive as a shared resource 
you'd type C: <Enter>. To set up your C:\APPS directory as 
a network resource you'd type C:\APPS <Enter>. 

5. Press Esc to exit each menu. You've now created the 
shared drive or directory resource. 

Copying a resource
==================
Once you've set up a disk drive or other type of device as 
a shared resource, you can copy the information from that 
resource and use it to create another one. You can copy 
resources from one server to another or, if you have 
multiple control directories set up, you can copy 
resources among them. For more information, see the file 
Crldir.txt.

1. At the DOS prompt, type NET_MGR <Enter>. Select Shared 
Resources Management <Enter>.

2. If you're copying only one resource, select it and 
press C. If you're copying multiple resources, simply 
press C. 

3. Set the fields in the Copy Resources screen: 

- The name of the selected resource appears in the From 
Resource field. If you're copying only this resource, 
leave the field at this setting. Otherwise enter the 
desired resource name to copy. 

- You may also use wildcard characters to specify more 
than one resource. For example, entering "*" selects all 
the server's resources and "@*" selects the server's mail 
resource and all of its printer resources. If you select 
more than one resource in this field, the To Resource 
field is automatically set to the From Resource setting 
and can't be edited.

- If you're copying only one resource, specify the name it 
will have after it's copied in the To Resource field.

- Specify the control directory where you want the 
resource(s) copied. This control directory can reside on 
another server. For more information on control 
directories, refer to the file Crldir.txt. 

- If the destination control directory is 
password-protected, select Password and type in the 
correct password <Enter>.

- If you want the copied resource(s) to replace any 
resource(s) with the same name(s) in the destination 
control directory, select Replace and press Enter to 
toggle the option to Yes. Otherwise, leave it at the 
default setting No. 

4. Press F10 to copy the resource(s), then press Esc to 
exit each screen.
 
Setting ACL rights for drive resources
======================================
As discussed under "Planning your drive resources" earlier 
in this file, Access Control List (ACL) rights allow you 
to limit the types of access that a user or group of users 
have to a shared resource. 

Important - Security rights on a disk resource are applied 
downward through the entire directory. For example, if you 
define C-DRIVE as pointing to C:\ and give full rights to 
that resource, a person keeps those rights at all levels 
within that drive.

Note - If you want to use ACLs to set up another level of 
server security, you must make sure the Access Control 
Lists option on the Security Configuration screen is 
enabled. You can check whether this is the case by 
selecting the Security and Send ID option on the Server 
Startup Parameters screen. 

When setting such rights for a large network, it's often 
easier to set Access Control List rights using ACL 
templates, as discussed under "Working with ACL templates" 
later in this file.

To set ACL rights for a drive resource:

1. If you want to assign a set of ACL rights to an ACL 
group, make sure that you've already created that group. 
If you haven't, follow the instructions under "Creating a 
new ACL group" in the file Accts.txt.

2. Go to a LANtastic computer. At the DOS prompt type 
NET_MGR <Enter>.

3. Select the Shared Resources Management option from the 
NET_MGR main screen <Enter>. Select the drive resource for 
which you'd like to set access rights <Enter>. The list of 
ACLs set up for this resource appears in the lower part of 
the screen, as shown. 

- If you haven't previously assigned any sets of ACL 
rights, only the default "*" ACL group appears, with all 
rights assigned by default. Highlight the "*" group and 
press Ins. 

- If you've previously assigned ACL rights to an account, 
ACL group or template, select an ACL entry in the Access 
Control List field at the bottom of the screen. The new 
ACL entry will be inserted before the selected entry. 
Press Ins. Make sure that you position the new ACL group 
so that there's no entry above it that matches the 
username(s) for which you're setting access rights. For 
more information, see "Placement order of wildcard 
accounts" in the file Accts.txt.

4. Type the name of the individual account, wildcard 
account or ACL group for which you want to set the ACL 
<Enter>. You can use wildcard characters in ACL names. For 
example, Sales-* would match any username that begins with 
SALES-. 

5. Type the letters that correspond to any access rights 
you want to enable or disable. These access rights are 
described in the next section. 

Hint - As you type the required letters, the corresponding 
access right appears automatically in the correct place on 
the access rights line. If you make a mistake and want to 
cancel an access right, type the letter again. The access 
right letter disappears and is replaced by a dash. 

When setting the ACLs, these keys perform the following 
functions:

Ins
---
Creates a new ACL.

Del
---
Removes an existing ACL.

F1
--
Provides Help information.

F3
--
Sets all options for the selected ACL. 

F4
--
Removes all options for the selected ACL. 

F9
--
Copies the ACL list to the clipboard.

F10
---
Pastes the ACL list from the clipboard.

Note - Because the default "*" ACL group allows all types 
of access to any individual or wildcard account on the 
network, delete all the rights that you don't want to give 
everyone for this drive resource.

6. Press Esc to exit each screen.

Access Control List rights
==========================
You can give an individual account, wildcard account, ACL 
group or template any combination of the Access Control 
List rights listed in this section on a shared resource 
such as a C: drive. As described under "Setting file-level 
security" later in this file, this can even be an 
individual file or directory. Unless the individual or 
wildcard account has the A (Super ACL) privilege, the user 
has only the types of access that you allow.

R - Read Access
---------------
Open files for reading.

W - Write Access
----------------
Write to files.

C - Create a File
-----------------
Create files. The user won't be able to write to these 
files if you don't enable the W (Write Access) privilege.

M - Make Directory
------------------
Create new subdirectories.

L - File Lookups (DIRs)
-----------------------
Display or search through directories.

D - Delete Files
----------------
Delete files.

K - Delete Directories 
----------------------
Delete subdirectories.

N - Rename Files
----------------
Rename files.

E - Execute Program
-------------------
Execute programs.

A - Change File Attributes
--------------------------
Change file attributes within a shared directory.

I - Indirect File
-----------------
Create and use indirect files within this shared 
directory. For more information, refer to your LANtastic 
printed and online manuals. 

Note - There's also a P (Physical Access) right that 
allows the user to connect to DOS devices directly, rather 
than through the despooler. Only experienced LANtastic 
users should be assigned Physical Access rights. 

Example sets of ACL rights
=========================
Here are some typical ACL rights for an individual 
account. If you want this user to have all access rights 
to a shared resource such as a server's disk drive, you'd 
set up an ACL list for that user like: 

BOB RWCMLDKNEA - -

If you want to restrict Bob to reading files from the 
drive and performing directory lookups (DIRs), you'd 
create an ACL list like this:

BOB R - - - L - - - - - - - 

For more sample accounts see the next section.

Important - Anyone with an account that has the A (Super 
ACL) privilege has full access to every resource on the 
server - regardless of whether you've assigned him ACL 
rights.

Working with ACL templates
==========================
You can set access rights for an entire group of users by 
entering a template name followed by a wildcard character 
("?" or "*"). The asterisk denotes that any number of 
characters may follow. 

Here are sample user group templates:

ADMIN-* 

$*

Any person whose individual account name (username) begins 
with either ADMIN- or $ (such as ADMIN-CLARA or $NED) has 
that group's access rights. 

You can also take advantage of the top-to-bottom scanning 
order to set ACL rights for any people who belong to a 
user group but who need different ACL rights. For example, 
if you wanted to assign more ACL rights to the username 
ADMIN-ROSIE, all you'd need to do is place her ACL before 
the account prefix in the list of ACLs. You could then 
allow her more access rights than the other users in her 
group:

ADMIN-ROSIE 	RWCMLDKNEA - -

ADMIN-*      RWCML - - NEA - -

In the above example, ADMIN-ROSIE has all access rights 
except the I (Indirect File) and P (Physical Access) 
privileges. Other usernames with the ADMIN- prefix, such 
as ADMIN-ROD or ADMIN-TAMRA, are allowed more limited 
rights. 

The same strategies for individual and wildcard accounts 
apply to ACL group accounts described above.

Here are some sample ACL rights assigned to individual, 
ACL group and wildcard accounts: 

BOB      RWCMLDKNEA - -

ED      RWCMLDKNEA - -

SALES      R - - - L - - - E - - -

ACCT      R - - - L - - - E - - -

* - - - - - - - - - - - - 

Setting file-level security
===========================
You can set access rights for individual directories and 
files within a shared drive resource. For example, you can 
make a file read-only or prevent people from renaming it. 
However, setting up file-level security will slow server 
response time.

Note - You can't re-enable file-level ACL rights that have 
previously been disabled at a higher level. In other 
words, you can't allow more access rights for a file 
template than you allow for the entire shared entire 
resource. For example, if a user is allowed only Read, 
Lookup and Execute (R, L, E) rights to the entire shared 
drive, she can't have Write (W) access to any files or 
directories within that shared resource.

To set up ACL security for an individual directory or file:

1. Check the ACL rights currently assigned to the 
individual account, wildcard account, ACL group or 
template to which you want to give file-level 
security. Make sure it has at least the same rights on the 
drive resource that you want to give it for the file. 
Here's how to check and change the ACL rights :

- On a LANtastic computer, type NET_MGR <Enter> at the DOS 
prompt. Select Shared Resources Management.

- Select the drive resource for which you'd like to set 
access rights <Enter>. The list of ACLs set up for this 
server appears in the lower portion of the screen. These 
rights and their single-letter abbreviations are listed 
earlier in this file.

- Select File Security, press Enter, then type in the full 
path of the file for which you want to set access rights 
<Enter>.

- If you need to assign more rights, follow the 
instructions under "Setting ACL rights for drive 
resources" earlier in this file.

- Choose OK in each window to go back to the Resources 
window.

You can include wildcard characters to set access for a 
group of files. For example, entering *.EXE would set 
access rights for all files with the .EXE extension in the 
specified path. You can also enter the special "..." 
wildcard, which will match any subdirectories contained 
within the directory. For example, if you create the 
template "\...\*.BAT" any batch file within the root 
directory or any subdirectory of the shared resource will 
have the access rights you set. The following is an 
example of protecting all executable files in the UTIL 
directory and all of its subdirectories from being 
overwritten, deleted, renamed or otherwise altered while 
still allowing everyone to execute the files:

\UTIL\...\*.EXE          *     R - - - L - - - E - - -

You can also use multiple wildcard characters to specify 
the names of directories and files. For example, the 
template "\WIN*\...\*.INI" indicates that any files having 
the INI extension within any directories that start with 
the letters WIN and all of their subdirectories will have 
common access rights. The following is an example of 
protecting all .INI files in the WIN* directory while 
granting full access to users belonging to the ADMIN group:

\WIN*\...\*.INI       ADMIN     RWCMLDKNEA - -

                      *              R - - - L - - - - - - -

2. Press Tab to move to the ACL Template screen. Position 
your cursor where you want to insert the new ACL template. 
Press Ins.

Important - Make sure you place the ACL so there's no 
entry above it that matches the username(s) for which 
you're setting access rights. 

3. Type the name of the account(s) for which you want to 
set the ACL. You can use wildcard characters in ACL names. 
For example, SALES-* would match any username that begins 
with SALES-. 

4. Type the letters that correspond to any access rights 
you want to set. 

When setting the ACLs, these keys perform the following 
functions:

Ins
---
Creates new ACL. 

Del
---
Removes ACL. 

F1
--
Provides Help information.

F3
--
Sets all options for selected ACL. 

F4
--
Removes all options for selected ACL. 

F9
--
Saves the ACL.

F10
---
Restores the ACL to the previously saved version, if any.

5. When you've finished setting the ACLs for the file 
template, press Esc to return to the File Templates 
screen. 

6. If you want to set access rights for more files and/or 
directories, press Ins and complete steps 3 - 5 for any 
additional file templates.

7. Press Esc to exit each screen. 

Copying and pasting an ACL
==========================
Once you've set up an Access Control List, you can copy 
it to any shared resource. That way you can quickly set up 
your server's resources. You can also copy the ACLs to 
resources in other control directories and even control 
directories on other servers. This saves time setting up a 
new server. 

1. At the LANtastic computer that has the resource with 
the ACL you want to copy and paste, type NET_MGR at the 
DOS prompt <Enter>.

2. Select the Shared Resources Management option from the 
NET_MGR main screen <Enter>. 

3. From the screen that appears, select the resource that 
has the ACL you want to copy and paste <Enter>.

4. Use the arrow keys to highlight one of the ACLs in the 
Access Control List section of the Detailed Information 
for Resource screen. (It doesn't matter which one.) Press 
F9 to copy all the ACLs for this resource.

5. Press Esc to return to the Resource Name screen. Select 
the resource to which you want to paste the ACLs <Enter>.

6. Use the arrow keys to highlight one of the ACLs in the 
Access Control List section of the Detailed Information 
for Resource screen. Press F10 to paste the copied ACLs 
into this resource. Press Esc to return to the DOS prompt.

New printer resource features
=============================
LANtastic 7.0's new printer resource features give you 
more choices and make printing easier. For example:

- You now can set up a printer pool resource that contains 
a prioritized list of other printer resources. When a user 
wants to print a job, LANtastic will send it to the first 
available printer in the list. Instructions for setting up 
this feature appear under "Creating a printer pool 
resource" later in this file.

- LANtastic 7.0 makes it easy to send jobs to 
Hewlett-Packard printers with a JetDirect network card 
installed or to printers connected to a JetDirect EX 
external unit. Instructions for setting up this feature 
appear in the next section.

- If your network includes a Central Station™ connectivity 
processor, you can now send print jobs to it faster and 
more easily than was previously possible using Central 
Station's CSPRINT program. Instructions for setting up 
this feature appear in the next section.

- You can now enable error notification on a printer 
resource, so the user who submitted the job and people 
with the O (Operator) privilege will get an error message 
if the job fails to print. Instructions for enabling this 
feature appear under "Creating a shared printer resource" 
below.

Creating a shared printer resource
==================================
LANtastic 7.0's new printer resource features give you 
more choices and make printing easier. The steps for 
creating a printer resource are similar to those for 
creating a drive resource. The major differences are that 
you must precede printer resource names with an "@" sign 
to distinguish them from drive resources, and you must 
specify the port to which the printer is attached. 

Note - This section explains how to set up a local 
printing resource. Instructions for setting up global 
resources appear later in this file. 

1. At the DOS prompt of a LANtastic computer, type NET_MGR 
<Enter>. From the Network Manager menu, select Shared 
Resources Management.

2. Press Ins and type in the name for the network printer 
<Enter>. 

An "@" sign must precede the name. The name, including 
the "@" sign, may contain up to eight characters, with an 
optional three-character extension. For example, you might 
type @PRINTER or @LASER.SML. 

Note - You can also create a shared printer pool resource 
as described under "New printer resource features" earlier 
in this file. Refer to "Creating a printer pool resource" 
later in this file for setup instructions. 

3. To create a parallel or serial port resource, select 
the LPT or COM port where the server's printer is attached 
<Enter>, and skip to step 6. 

You can also select HP JetDirect, CS Print, or Printer 
Pool options. Refer to the LANtastic Installation and 
Setup Manual for full instructions on setting up an HP 
JetDirect printer. Refer to step 4 to create a CS Print 
resource. To create a printer pool resource, refer to 
"Creating a printer pool resource" later in this file for 
instructions. 

4. To create a CS Print resource, select CS Print <Enter>. 
The LANtastic 7.0 CS Print option provides faster printing 
to printers attached to a Central Station™ than was 
previously possible.

- Enter the network name of the Central Station that has 
the printer attached<Enter>. You can use up to 15 
characters.

- Enter the adapter number (0 to 253) that you're using to 
connect to the Central Station. Typically this will be 
adapter zero <Enter>. Select the Central Station port that 
the printer is attached to <Enter>, and skip to step 7.

5. Select the new printer resource <Enter>. 

If you selected an LPT port for the printer resource, the 
Detailed Information screen appears. The fields on the 
screen are different depending on the type of printer 
resource it is for. For information about each of the 
fields in these screens, refer to the next sections. 

6. Make sure the settings in the menu are correct for your 
printer. If not, select the item you want to change, press 
Enter, and type in the new information. (If you aren't 
certain about a setting, refer to your printer manual.)

7. Select the Access Control List account to modify any of 
the access rights for this resource. See "Setting ACL 
rights for drive resources" earlier in this file for more 
information on Access Control Lists.

8. Press Esc to exit each screen.

Important - If you set up either HP JetDirect or CS Print 
resources, you must enable either HP JetDirect or CS Print 
support in the Server Startup Parameters screen's Printing 
Configuration display. See the file Cfgsrv.txt for 
instructions.

Note - If you're using Microsoft Windows or another 
application that requires a special driver to use a 
specific type of printer, you must set up that printer 
according to your Windows or application program 
instructions. Having the correct driver installed on the 
print server usually isn't sufficient. You must install 
the correct driver on each workstation that uses the 
printer. For more information on network printing, refer 
to "Printing" in the file Cfgsrv.txt and the file 
Mngprt.txt. 

Options in the Detailed Information screen
==========================================
For each of the options, you'll see the name of the field 
and the default setting for that field in brackets, 
followed by the valid range of values and a description.

Description
-----------
Allows you to include a brief description for this shared 
resource. Descriptions can be up to 64 characters long.

Output Device
-------------
[LPT1] LPT1, LPT2, LPT3, COM1, COM2, COM3, COM4, 
HP JetDirect, CS Print, Printer Pool, Remote, Keyboard - 
Lets you specify the parallel or serial port or other 
location where the printer or device is installed. This 
selection is used to form the name of the queue which will 
actually receive print jobs.

If the output device is HP JetDirect, its corresponding 
ID number and adapter number is also specified.

If the output device is CS Print, its corresponding ID 
number, adapter number and communication port (LPT1, COM1, 
COM2, PC or AUX) is also specified.

Error Notification
------------------
[Disabled] Enabled, Disabled - Specifies that the user who 
submitted the current job, as well as users with the O 
(Operator) privilege, will receive notification of events 
that require intervention, such as the printer running out 
of paper. 

Notification T/O
----------------
[0] 0-3600 - Sets the number of seconds before 
notification is sent. The Notification option must be 
enabled to set this field. 

Job Notification
----------------
[Disabled] Enabled, Disabled - When enabled, notifies you 
when your print job is finished. This option overrides the 
command NET LPT/DISABLE NOTIFY.

Banner Page
-----------
[Disabled] Enabled, Disabled - Places a banner page before 
each print job. 

Banner Paper Width
------------------
[0] 0-255 spaces - Sets the paper size for the banner 
page. Typical values are 80 and 132 characters. If you're 
using standard letter-size paper, you don't need to set 
this field, since it's set to 80 internally.

Form Feeds
----------
[Disabled] Enabled, Disabled - Instructs the printer to 
add a form feed at the end of each job. This advances the 
paper to the top of the next page after each print job. 
This can help prevent mixing print jobs. When using setup 
and cleanup files, the form feed is sent before the 
cleanup file. 

Immediate Despooling
--------------------
[Disabled] Enabled, Disabled - Allows the server to begin 
despooling a print job before all the information has been 
written to its disk. This is especially useful if you'll 
be printing large files, since you won't need to wait for 
the entire file to be spooled before it begins printing.

Despool Timeout
---------------
[Disabled] Disabled, 0-3600 seconds - If you enable 
Immediate Despooling, this option sets the amount of time 
that the printer can sit idle before the server aborts an 
immediately despooling file and reschedules it for normal 
despooling. Guidelines for setting this value appear in 
the file Mngprt.txt. 

Lines per Page
--------------
[Disabled] Disabled, 0-255 lines - Sets the number of 
lines before a form feed is automatically inserted. The 
default value (zero) disables this function. This is 
useful for formatting text-only (ASCII) files. Don't 
enable this option if you'll be printing binary files such 
as graphics or PostScript® files.

Tab Width 
---------
[0] 0-255 spaces - Sets the number of spaces between tab 
stops that the despooler will emulate each time a tab is 
found in a print job. This is useful when using printers 
that don't support tabs. Don't enable this option if 
you'll be printing binary files such as graphics or 
PostScript® files.

Characters/Second
-----------------
[0] 0-32767 CPS - Sets the minimum number of characters 
per second that the despooler will attempt to send to the 
printer. If the server is performing other network tasks, 
it will give higher priority to the print job in an effort 
to print the number of characters per second that you set. 
If you set a value higher than the computer or the printer 
can perform, the job is printed at the highest possible 
speed.

Setup Delay
-----------
[Disabled] Disabled, 0-3600 seconds - Specifies the number 
of seconds the despooler will wait between sending the 
setup string to the printer and sending the actual data to 
be printed. This gives the printer time to initialize the 
setup string.

Cleanup Delay
-------------
[Disabled] Disabled, 0-3600 seconds - Specifies the number 
of seconds the despooler will wait after sending the 
cleanup file. This gives the printer time to initialize 
after the cleanup file is sent.

Edit Setup String 
-----------------
Allows you to edit the setup string that the despooler 
will send to the printer at the start of a print job. You 
can include any valid parameters for your printer. You 
can also specify a setup file and cleanup file combination 
with this option. If you use the setup string, you can't 
use setup and cleanup files and vice versa. If you'll be 
using setup strings or setup and cleanup files, read the 
section describing setting up the two sample printer 
resources at the end of this file.

Serial printer options in the Detailed Information screen
=========================================================
This section provides information on each of the fields at 
the bottom of the Detailed Information screen for a serial 
printer. For each option, you'll see the name of the field 
and the default value for that field in brackets, followed 
by the valid range of values and a description.

Port Access
-----------
[Hardware] Hardware or INT 14 - Specifies the method used 
to communicate with the server's serial port. Use the 
Hardware setting unless an application requires that you 
use INT 14 redirection. If you select INT 14, fewer baud 
rate options appear and the Mark Parity option is 
unavailable.

Baud Rate
---------
[9600] 110-115200 bps - Allows you to set the baud rate 
for the printer resource. The baud rate is the number of 
bits per second that will be sent through the server's COM 
port to the printer. 

Word Length
-----------
[8] 7 or 8 bits - Allows you to select either a seven-bit 
or eight-bit word length. Data is sent to a serial printer 
in groups of bits called words. Most serial devices use an 
eight-bit word length. 

Stop Bits
---------
[1] 1 or 2 bits - Adds one or two bits to the end of a 
word. 

Parity
------
[None] None, Odd, Even, Mark - Sends a parity bit after a 
character's data bits. This option is used to detect 
transmission errors in serial communications. The actual 
value of the parity bit is based upon the type of parity 
checking used and the character's data bits. No [None] 
parity means there's no parity bit appended and thus no 
parity checking. Even parity results in the parity bit 
being set to zero or one so that adding the data bits and 
the parity bit yields an even number. Odd parity means 
that adding the data bits and the parity bit yields an odd 
number. Mark parity results in the parity bit always being 
set to one. This is often used when sending 
seven-bit-length words to devices that require an 
eight-bit word length.

Flow Control
------------
[Hardware] None, Hardware, Xon/Xoff - Allows you to select 
whether the hardware or software will control the flow of 
bits between your computer and the printer. Most serial 
devices use hardware flow control. Software flow control 
is accomplished by sending a series of XON and XOFF 
characters through the serial port to indicate whether the 
printer is ready for more data. If you select the None 
option, the bits will be sent through the serial port to 
the printer with no flow control. If the data is sent 
faster than the printer can process it, characters will be 
lost.

Using the printer setup string
==============================
The printer setup string allows you to set up different
print resources for the same printer. Each of these 
resources can use a different print mode, such as 
condensed or landscape.

The printer setup string is a group of binary or ASCII 
control characters that's sent to a printer at the start 
of a print job. Consult your printer manual for more 
information. If you're using the setup string, you can't 
use setup and cleanup files. 

1. At the DOS prompt of the LANtastic computer that has 
the printer resource for which you want to use setup 
strings, type NET_MGR <Enter>. Select Shared Resources 
Management <Enter>.

2. Select the printer resource that will use a setup 
string <Enter>.

3. Select Edit Setup String <Enter>. 

4. Select Setup String <Enter>. A screen appears where you 
can enter the control codes for this printer resource. A 
one-character entry designates ASCII and a two-digit entry 
designates a hexadecimal value.

5. Press Ins and type the first control code <Enter>. It 
can be either a two-digit hexadecimal number or a single 
ASCII character. Type in any additional codes that you 
want to enter. The setup string may include up to 64 
codes. 

6. When you've finished entering control codes, press Esc 
to stop the input mode, then press Enter to save your 
changes.

7. Press Esc in each screen to return to the Detailed 
Information screen.

8. In the Setup Delay field, enter the number of seconds 
the despooler will wait between sending the setup string 
to the printer and sending the actual data to be printed 
(from 0 through 3600 seconds). This is useful for printers 
that need time to execute the setup string. The value you 
specify depends on the length of time it takes to 
initialize your printer.

9. Press Esc to exit each screen.

Using setup and cleanup files
=============================
Setup and cleanup files are text files that contain 
decimal numbers, hexadecimal numbers or ASCII characters 
like the setup string described in the previous section. 
However, these types of files allow you to include more 
control codes than the 64-character limit for the printer 
setup string. First the setup file is sent to initialize 
the printer for a specific mode of printing. After the 
job is printed, the cleanup file is sent to reset the 
printer, usually to its original mode of printing.

If you haven't created the printer resource to use setup 
and cleanup files, complete the steps listed at the 
beginning of "Creating a shared printer resource" earlier 
in this file.

To use setup and cleanup files:

1. At the LANtastic computer that has the printer resource 
for which you want to use setup and cleanup files, use a 
text editor to create the setup and cleanup files. These 
files must contain only valid control characters for your 
printer. Consult your printer manual for information. 

2. At the DOS prompt, type NET_MGR <Enter>. Select Shared 
Resources Management <Enter>. 

3. Select the printer resource that will use a setup file 
<Enter>.

4. Select Edit Setup String <Enter>. 

5. Set the Setup Filename and Cleanup Filename options

- Select Setup Filename <Enter>.

- Enter the full DOS path of the setup file you created in 
step 1 <Enter>. 

- If a setup string already exists, you'll be prompted to 
confirm that you want to overwrite that setup string 
<Enter>.

- Select Cleanup Filename <Enter>.

- Enter the full DOS path of the cleanup file you created 
in step 1 <Enter>. 

- Press Esc to exit to the Detailed Information screen.

Note - If this resource will be serviced by a remote 
printer server, you must enter a network path to the 
setup and cleanup files such as 
\\servername\C-DRIVE\SETUP.TXT and 
\\servername\C-DRIVE\CLEANUP.TXT instead of the DOS path.

6. Set the Setup Delay and Cleanup Delay options:

- Select Setup Delay <Enter>.

- Enter the number of seconds the despooler will wait 
between sending the setup file to the printer and sending 
the actual data to be printed (from 0 through 3600 
seconds) <Enter>. This is useful for printers that need 
time to initialize the setup string. 

- Select Cleanup Delay <Enter>.

- Enter the number of seconds that the despooler will wait 
between sending the cleanup string to re-initialize the 
printer and beginning the next print job (0-3600 seconds) 
<Enter>.

7. Press Esc to exit each screen.

Creating a printer pool resource
================================
A printer pool resource is an ordered list of printer 
resources. Print jobs sent to the printer pool are printed 
on the first available printer listed in the pool. Pooled 
printers can include parallel ports, serial ports and 
network-ready printers. Printer pools should be created 
with printers that accept equivalent formats (e. g., HP 
LaserJet-compatible). You can't use global printer 
resources in printer pools.

1. If you haven't already done so, set up each printer you 
want to include in the pool as a shared resource. 
Instructions appear under "Creating a shared printer 
resource" earlier in this file.

2. At the DOS prompt, type NET_MGR <Enter>. Select Shared 
Resources Management <Enter>. 

3. Press Ins and type in the name for the printer pool 
<Enter>. 

An "@" sign must precede the name. The name, including 
the "@" sign, may contain up to eight characters, with an 
optional three-character extension. For example, you might 
type @POOL or @LASER.DIP. 

4. Select Printer Pool <Enter>.

Select a printer from the list of Available Pool Resources 
<Enter>. The resource appears in the Pool Resources 
screen. A maximum of eight devices can be selected. You 
can use the Tab key to switch between the active screens, 
enabling you to place pool devices in a specific order of 
priority in the Pool Resources screen. For example, when 
you activate the Pool Resources screen and highlight a 
current resource, any resource you then add from the list 
of Available Pool Resources will be placed above that 
selection. When the Pool Resources screen is active, you 
can use the Delete key to delete resource items from it. 
Press F2 to save your selections, then press Esc.

5. Select the new printer resource <Enter>. The Detailed 
Information screen appears. 

- Select the Descriptions field <Enter> to enter a 
description for the printer pool resource.

- To add or delete printer resources from the pool 
resource, select Output Device <Enter>. Select Printer 
Pool from the Output Device screen <Enter>. You can now 
select resources to add to or remove from the pool. Press 
F2 to save your selections.

6. Select the Access Control List account to modify any 
of the access rights for this resource. See "Setting ACL 
rights for drive resources" earlier in this file for more 
information on Access Control Lists.

7. Press Esc to exit each window.

Note - If you're using Microsoft Windows or another 
application that requires a special driver to use that type
 of printer, you must set up that printer according to 
your Windows or application program instructions. Having 
the correct driver installed on the print server is 
usually not sufficient. You must install the correct 
driver on each workstation that uses the printer.

Setting up a global resource
============================
A global resource is defined on one server but points to 
resources existing on another server. That way, users can 
log in to only one server to use multiple servers' 
resources. This is similar to "Domain" resources found 
in other network operating systems. A global resource 
improves response time, since workstation requests don't 
need to be routed through two servers. It also makes 
managing your network resources easier, since there's 
only one set to manage.

Note - To access a global resource, a user must have a 
valid account and sufficient access rights to the server 
that's actually sharing the resource.

Creating a global resource 
--------------------------
1. At the DOS prompt, type NET_MGR <Enter>. Select Shared 
Resources Management <Enter>.

2. Press Ins and type the name for the global resource 
<Enter>. 

- For example, you might type DATA for a global drive 
resource. 

- For a global printer resource, an "@" sign must precede 
any printer resource names. The resource name including 
the "@" sign may be up to eight characters long, with an 
optional three-character extension. For example, you might 
type @GLOBAL.

3. Type in the full network path to the server and its 
shared drive or printer resource <Enter>. Here are sample 
drive and printer network paths:

\\SERVER1\C-DRIVE
\\SHIRLEY\APPS
\\SERVER1\@PRINTER
\\JUDY\@HP3

4. Select the Access Control List account to modify any of 
the access rights for this resource. See "Setting ACL 
rights for drive resources" earlier in this file for more 
information on Access Control Lists.

5. Press Esc to exit each screen.

Creating a shared keyboard or screen resource
=============================================
By default, Setup creates resources for a server's 
keyboard and screen. This allows you to view the server's 
screen from your own computer and send commands to be run 
on that server. This has the same effect as going to the 
server and typing the command at its keyboard. 

If you've upgraded from a previous version of LANtastic or 
have deleted these resources, you'll need to create them 
to use these features.

1. At the DOS prompt, type NET_MGR <Enter>. Select Shared 
Resources Management <Enter>.

2. Press Ins and type in a name for the server's keyboard 
or screen resource such as @KEYBD or @SCREEN <Enter>. The 
names of keyboard and screen resources must begin with the 
"@" symbol and must be no longer than eight characters 
with a three-character extension. 

3. Under the Direct section of the Output Device screen, 
select either Screen or Keyboard <Enter>. 

4. The new resource appears on the list of server 
resources. Select it <Enter>.

5. Enter a description of the resource (optional) <Enter>.

6. Set the Mode field to the desired setting. Pressing the 
Enter key toggles the option between Binary and ASCII.

Binary is the default setting. It lets you type or see 
characters and attributes, such as their shift state 
(case) and color. Generally you should use this setting 
for screens.

Select ASCII if you'd like to send or see the characters 
without any attributes. This is useful if you want to copy 
a server's screen to a file.

Note - The Watched Icon option is enabled by default. This 
causes an asterisk to appear on the server's screen 
whenever a user is viewing the server's monitor remotely. 
If you don't want a user at the remote server to know when 
his computer is being monitored, disable this option. 

7. Select the Access Control List account to modify any of 
the access rights for this resource. See "Setting ACL 
rights for drive resources" earlier in this file for more 
information on Access Control Lists.

8. Press Esc to exit each screen.

Creating a spooled keyboard resource
====================================
Creating a spooled keyboard resource allows you to send a 
batch file to a server. The batch file will be executed on 
the server at a time you specify. This is equivalent to 
executing multiple NET RUN commands on that server, and 
makes it convenient to schedule server backups, since you 
can send the backup batch file to the server's spooler and 
have it executed after office hours.

By default INSTALL creates a spooled keyboard resource 
called @BATCH. If you upgraded from a previous version of 
LANtastic or deleted this resource, you'll need to create 
one to use this feature.

1. At the DOS prompt, type NET_MGR <Enter>. Select Shared 
Resources Management <Enter>.

2. Press Ins and type in a name for the resource <Enter>. 
The names of keyboard and screen resources must begin with 
the "@" symbol and must be no longer than eight characters 
with a three-character extension. For example, you might 
type @BATCH. 

3. Under the Spooled section of the Output Device screen, 
select Keyboard <Enter>. 

4. The new resource appears on the list of server 
resources. Select it <Enter>.

5. Enter a description of the resource (optional) <Enter>.

6. The Mode field is set to ASCII by default. Generally 
you should use this setting. ASCII allows you to send 
characters without any attributes. 

Pressing the Enter key toggles the option between ASCII 
and Binary. Selecting Binary lets you send characters with 
attributes such as shift state and color. 

7. Select the Access Control List account to modify any of 
the access rights for this resource. See "Setting ACL 
rights for drive resources" earlier in this file for more 
information on Access Control Lists.

8. Press Esc to exit each screen.

For detailed information on using a spooled keyboard 
resource, refer to the file Mngsrv.txt. 

Creating a CD-ROM or other non-DOS drive resource
=================================================
During software installation, LANtastic automatically 
created a resource for each CD-ROM drive that it detected 
on a server. However, if you attach a CD-ROM after 
installing LANtastic, you'll have to set it up as a shared 
non-DOS drive resource.

If you want to share a WORM or other type of non-DOS 
drive, it also must be set up as a shared resource. 
However, non-DOS drive resources provide no file and 
record locking for users accessing the disk through the 
LANtastic server. 

To create a CD-ROM or other non-DOS drive resource:

1. Refer to the documentation for the type of non-DOS 
drive you're setting up. Then take any necessary steps in 
addition to the ones described in this section. For example:

- Follow the instructions in the file Expand.txt for 
adding a CD-ROM drive to your network before you 
follow the steps below.

- Before setting up a WORM drive as a non-DOS resource, 
you must install its device driver in your CONFIG.SYS 
file. You must then set up the drive as a shared resource 
as instructed below. Once this is done, users can log in 
to this server and access the WORM drive.

2. Complete steps 1 - 4 for creating a shared drive 
resource as instructed under "Creating a shared disk or 
directory resource" earlier in this file.

3. At the NET_MGR main menu, select Shared Resources 
Management, press Enter and select the new resource 
<Enter>. 

4. Select Disk Drive Type <Enter>, then select the type of 
drive you're using (CD-ROM, WORM or other) <Enter>. If a 
message about using non-DOS disks appears, take a moment 
to read it then press Esc.

5. Press Esc to go back to the NET_MGR main menu.

6. Select Server Startup Parameters. Enter the full path 
to LANtastic's SERVER.EXE program (usually 
C:\LANTASTI\SERVER.EXE) . In the Server Startup Parameters 
screen, make sure Non-DOS Disk Support is enabled. If it 
isn't, use the arrow key to select it, then give it a 
setting of 100. For further information about this 
setting, see the file Cfgsrv.txt. 

Note - When using a shared non-DOS disk connection, you 
can't use wildcard characters such as "?" and "*" with the 
DOS DEL and REN commands. Also, if you can't see all the 
files in a shared drive when you use the DOS DIR command, 
increase the Non-DOS Disk Support setting in the Server 
Startup Parameters screen. Steps appear in the file 
Cfgsrv.txt. 

NetWare drive
=============
In most cases you shouldn't use this option to connect to 
a NetWare computer. To connect to a NetWare server and use 
its resources, use LANtastic's NCP support feature, as 
described in the file Clent.txt.

OS/2 HPFS drive
===============
In most cases you shouldn't use this option to connect to 
an OS/2 computer. To set up an OS/2 HPFS drive as a shared 
resource, use Artisoft's LANtastic for OS/2 product for 
true peer-to-peer networking with a native OS/2 interface. 
See your LANtastic dealer or call Artisoft Sales and 
Customer Service for more information.

Sample parallel printer resource setups
=======================================
Any mode of printing that your printer supports can be set 
up as a shared resource. Consult your printer manual for 
information on control codes for your printer.

Example 1 - Condensed printing resource
=======================================
This sample resource uses a printer setup string to 
instruct the printer to output characters in condensed 
mode. For more information, see "Using the printer setup 
string" earlier in this file.

To create a condensed printer resource:

1. Complete the steps in "Creating a shared printer 
resource" earlier in this file. 

2. Select Shared Resources Management from the NET_MGR 
screen.

3. Select the resource in the Resource Name => Local 
Path/Device screen <Enter>.

4. Select Edit Setup String <Enter>. 

5. A screen appears with the field Setup String 
highlighted <Enter>. 

6. A dialog box appears where you can enter the control 
codes for this printer resource. Press Ins and type in 
your printer's condensed mode control codes, then press 
Enter to confirm. 

7. Press Esc to exit each screen.

The printer will now output any jobs sent to this print 
resource in condensed mode. The printer will then be reset 
when a user issues a request to a different printer 
resource.

Example 2 - PostScript printer resource
=======================================
The printer in this example has a PostScript cartridge 
installed. This means that it's possible to create a 
shared network PostScript printing resource. You can use 
the same concepts presented in this section to create 
printer resources using other add-on cartridges.

The previous example used the setup string to initialize 
the printer. In this sample, the printer must perform a 
complete reset. Because of this, the PostScript printer 
resource uses setup and cleanup files as described in 
"Using setup and cleanup files" earlier in this file. The 
cleanup file ensures that once a special mode of printing 
is finished, any other print resource can be used.

To create a PostScript printer resource:

1. Complete the steps under "Creating a shared printer 
resource" earlier in this file. 

2. Select Shared Resources Management from the NET_MGR 
screen <Enter>. 

3. Select the new resource in the Resource 
Name => Local Path/Device screen <Enter>.

4. Select Edit Setup String, press Enter, then select 
Setup Filename <Enter>. 

5. Type in the full DOS path of the file containing the 
control characters for initializing the printer for 
PostScript printing <Enter>. Press Enter again to confirm 
that you want to overwrite the default setup string.

6. Select Cleanup Filename, press Enter, then type in the 
name of the file containing the control characters to 
re-initialize the printer for normal printing <Enter>. 

7. Set the setup and cleanup delays for the printer 
resource. This sample uses a 60-second delay for both 
options.

- Select Setup Delay, press Enter, then type in the number 
of seconds the despooler will wait between sending the 
setup string or setup file to the printer and sending the 
actual data to be printed (from 0 through 3600 seconds) 
<Enter>. 

- Select Cleanup Delay, press Enter, then type in the 
number of seconds that the despooler will wait between 
sending the cleanup string to re-initialize the printer 
and beginning the next print job (0-3600 seconds) <Enter>.

8. Press Esc to exit each screen.

Note - Consult your printer manual for proper codes for 
your cartridge. 

Once you've completed the steps above, the resource is 
created. When a user sends a job to this print resource, 
the printer resets itself for PostScript printing and 
doesn't allow output to any non-PostScript print resources 
set up on this server. After the job is finished, the 
cleanup file resets the printer, and people may use other 
non-PostScript print resources. 

Sample serial printer setup
===========================
The example below is for a plotter printer. It's a fairly 
simple setup with very few special printer parameter 
settings. If desired, a more complicated printer resource 
can be created for this printer using the setup string, or 
setup and cleanup files as shown in "Sample parallel 
printer resource setups" earlier in this file. 

1. Complete the steps listed in "Creating a shared printer 
resource" earlier in this file. 

2. Select Shared Resources Management from the NET_MGR 
screen <Enter>. 

3. Select the new resource in the Resource Name => 
Local Path/Device screen <Enter>.

4. Refer to your printer's manual for the correct Port 
Access, Baud Rate, Word Length, Stop Bits, Parity and Flow 
Control settings for your serial printer.

5. Press Esc to exit each screen.
